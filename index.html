자기소개서

성장과정

생활코딩을 보고 웹 페이지쯤은 쉽다고 생각을 하던 때에 인터넷 브라우저에 검색을 해서 “내가 만든 웹페이지를 링크로 들어오려면 어떻게 해야 할까?”라는 궁금증을 가지고 인터넷을 열심히 찾고 있었습니다. 지금까지 해왔던 것은 front-end 쪽이고 back-end 부분을 다루고 도메인 등록을 해야 브라우저 검색을 하면 볼 수 있다는 것을 알게 되었습니다. 유튜브로 코딩을 먼저 접했던 저는 유튜브에는 back-end 관련된 정보가 굉장히 적다는 것을 알게 되었고 혼자서는 아주 오래 걸리겠다 싶어서 국비 교육을 듣게 되었습니다. 국비 교육에서는 자바에 대한 기초와 Spring Boot로 서버를 구동하고 mySql을 연결해서 로그인도 되는 간단한 게시판을 만들었습니다. 수료하기 전에 팀 프로젝트를 총 3번 했습니다. 첫 번째 프로젝트는 아스날 펜 페이지를 만들었습니다. 실시간 점수와 통계를 구현하고 싶어서 api를 찾고 있었는데 모두 유료라서 셈플 데이터를 사용하였습니다. 메인 페이지에는 아스날 관련 최신 뉴스를 입력하고 싶어서 네이버 뉴스검색api를 이용하여 xml파일로 변경해서 저장 후 불러오는 식으로 진행했습니다. 두 번째 팀 프로젝트로 안드로이드 웹뷰를 이용한 피자 주문앱 클론 웹앱페이지를 만들었습니다. 마지막 프로젝트는 부산 명소를 소개함과 숙소를 임시로 만들어서 DB에 저장 후 예약하는 기능과 명소를 즐겨찾기 해서 여행경로를 계획 할 수 있는 웹페이지를 만들었습니다. 지도가 필요 할것 같아서 카카오 지도 api를 이용하여 만들었습니다.

사회활동 사항

[전기과 졸업 후 활동 사항]
졸업 후 3개월간 HTML과 CSS, JavaScript를 독학으로 공부하면서 개인 프로젝트를 몇 개 하였고 그 후 국비 교육을 들으면서 Java, JSP, Rest Api, Ajax, Paging, BootStrap 5, ThymeLeaf를 배웠습니다. 또한 Spring boot 팀 프로젝트 2회, android studio를 이용한 팀 프로젝트 1회, Java 개인 프로젝트 3회, Jsp 개인 프로젝트 1회를 하였습니다. 팀 프로젝트는 Spring boot로 서버를 구현하고 MVC 패턴을 사용하였습니다.

[4번의 팀 프로젝트]
먼저, 졸업 후 3번의 프로젝트는 스프링 부트에서 MVC 패턴을 이용한 Back-end 작업과 BootStrap 5 를 이용한 Front-end 작업을 하였습니다. 두 번의 프로젝트 모두 팀장을 맡았지만 팀원들의 참여율이 아주 좋았기에 팀장으로서 한 일은 회의 언제 할지 시간을 정한 것 뿐이였습니다. 회의의 내용은 주제 선정, 스토리 보드 작성, DB 설계, 요구사항 정의서 작성, 역할 정하기로 하였습니다. 첫 프로젝트에서 제가 맡은 부분은 로그인과 마이페이지 회원가입 등 유저에 관한 부분과 자유게시판 부분이였습니다. 로그인을 하기 전에 회원 가입을 하는 과정에서 중복 확인 기능을 구현 하는데에 있어서 난관에 부딪혔습니다. 처음 생각 했을때 중복 확인을 하기 위해서 DB에 같은 아이디가 있는지 확인을 해야 했습니다. 그것을 쿼리문에 select count(*) from table_name;으로 찾은 후 결과를 보는 것 까지는 했는데 페이지가 자꾸 이동 돼서 원인을 알아보았습니다. 페이지는 그대로 있고 정보만 주고받기를 원한다면 ajax 통신을 사용해야 한다는 것을 알게 되었습니다. 하지만 html의 script태그 안에 ajax만 사용한다고 되는 것은 아니였습니다. controller 쪽에 ajax에서 보낼 url을 @RequestMapping(value=url) 이 부분 위에 따로 적어주어야 하는 것이 있었습니다. 받고 데이터를 주고받는다는 뜻인 @ResponseBody가 있어야 한다는 것을 구글링을 통해 알게 되었습니다. 두번째 프로젝트에서는 웹 뷰를 이용한 피자 주문 앱 프로젝트였습니다. 모든 구현을 웹으로 다 하고 안드로이드 가상 기기(android emulatere)로 확인을 하는데 카카오 맵이 동작을 하지 않았습니다. 회원 가입을 위해 넣어 놨던 다음 우편번호는 잘 동작이 되는데 카카오 맵만 동작이 되지 않았습니다. 구글링을 통해 알아본 정보로는 유즈퍼미션을 줘야 한다고 했습니다. 하지만 구글링을 통한 정보를 바탕으로 수정을 해도 동작이 제대로 되지 않았고 동료들이 알아본 정보를 종합하니 안드로이드 버전이 업데이트 되면서 보안이 강화되어서 그 보안 관련된 부분을 공부해야 했습니다. 발표시간이 다가오고 있었기에 해결하지 못했지만 다른 api를 사용해봤다는 점에서 굉장히 만족스런 프로젝트 경험이였습니다. 세 번째 프로젝트에는 첫 번째 프로젝트와 마찬가지로 Spring boot를 사용했습니다. 패턴도 같고 사용한 DB도 MySql로 같습니다. 다른 점은 주제입니다. ”부산 여행 오소“라는 제목이고 부산에 유명한 곳과 그 주변의 숙소를 찾아주는 웹 애플리케이션입니다. 대학 때는 전기과에 맞는 프로젝트를 해야 해서 아두이노와 압전 소자를 이용한 전기 발전량 확인을 LED로 할 수 있는 주제로 했습니다.

[대학 졸업 전 활동 사항]
복학 후 학교를 좀 더 다니다가 휴학을 했습니다. 휴학기간 동안 영상편집을 공부했습니다. 가족들이나 친구들과 놀러 갔을 때 찍은 사진이나 영상들을 보관하기 위해 영상편집을 취미로 하다가 알고지내던 형이 유튜브를 하고 싶다고 해서 몇 개 편집해주고 용돈을 조금 벌었습니다. 중학생 때는 배드민턴 동아리를 다니면서 몸을 움직이는 것이 그 당시에는 가장 재밌었습니다. 고등학교에 올라가서는 체육관련 동아리가 없는 학교여서 만들고자 했으나 선생님들이 허가를 내어주지 않았습니다. 그 당시 유지되고 있는 동아리 ’사이언스 24‘ 라는 과학동아리에 들어가서 개구리 해부부터 자석의 회전 속력에 따라 얼마만큼의 전기가 생성되는지에 대한 건전지 충전 실험 등 여러 실험을 하였습니다. 대학교를 가서는 개인적인 취미로 아두이노로 유튜브에 나오는 몇가지 재밌는 물건을 만들었습니다. 작은 전동기를 사용해서 계란 공예에 쓰이는 전동기로 만든다던가, pcb 기판에 납땜하려고 하는데 인두기가 고장나서 전기가 발생하면 열도 같이 발생한다는 원리를 이용해서 만든 인두기로 납땜을 시도했습니다. 아두이노 덕에 c언어에 관심을 가졌고, 그 계기로 국비 지원 교육을 들었습니다.
